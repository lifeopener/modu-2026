## 1. 핵심 기획 의도(Line 23) 완벽 동기화 요약

기획서의 가장 중요한 선언인 **"억압적 UI 배제 + 실시간 웹데이터(뉴스) 기반 실무 성과 창출 + 5만 원 미만 예산 + 바이브 코딩 최적화"**가 모든 기술 문서에 어떻게 관철되었는지 정리한 매트릭스입니다.

| 핵심 기획 (Value Proposition) | Vibe Coding 명세 (`03` 문서) 반영 내역 | Human Dev 명세 (`04` 문서) 반영 내역 |
| :--- | :--- | :--- |
| **1. 화면 통제 / 억압적 UI 전면 배제** | `AmberGlowOverlay` 컴포넌트 및 `Zustand` 블러 상태 전면 삭제. 시스템 프롬프트에 "강압적 수단 금지" Rule 명시. | NFR-001 항목 신설: "어떠한 경우에도 화면 전체 블러나 매매 앱 전환 방해 로직이 동작하지 않아야 함." (위반 시 QA Reject) |
| **2. 실시간 웹데이터 기반 성장 코칭** | Vercel AI SDK의 `Tool Calling` 로직 추가. `getRealTimeNews` (Tavily API 연동) 코드를 Deno 엣지 함수 내부에 직접 하드코딩. | 외부 API 아키텍처에 Search API 블록 추가. RAG 검색과 Web Search가 동시 실행되는 실전 시퀀스 다이어그램 신설. |
| **3. B2B 교육/실무 큐레이션 (Nudge)** | 시스템 프롬프트 지시어 보강: "문맥 분석 후 도움이 될 때 자연스럽게 프리미엄 자료(B2B)를 마크다운 링크로 제안하라." | 팝업 강제 노출 테스트 케이스 삭제 → 채팅창 내 B2B 링크 렌더링 및 '클릭 전환율(CTR ≥ 8%)'을 A/B 테스트 지표로 복원. |
| **4. MVP 운영 예산 5만 원 미만 통제** | `gpt-4o-mini` 모델 및 `maxTokens: 250` 강제. `axios` 대신 `fetch` 사용. 서버비 0원을 위한 Supabase Edge Function 구조 강제. | 월 예산 한도 임계치 도달 시(80%), 무거운 API인 '실시간 웹 검색 도구'를 우선 비활성화하는 예산 방어 로직(Fallback) 추가. |
| **5. 바이브 코딩 초보자 최적화 (Zero Hallucination)** | **"Rule Zero"** 선언. Pinecone 등 유료 DB 설치 불가 명시. `pgvector` 기반 DDL 쿼리와 TypeScript 훅 완성본 복사/붙여넣기 제공. | 인간 개발자/QA가 AI의 산출물을 100% 빈틈없이 검증할 수 있도록 데이터 타입, 에러 타임아웃(2500ms), 엣지 케이스 테이블 완성. |

## 2. 기존 아키텍처 대비 주요 변경점 (History)

*   **[제거] Amber Glow 화면 블러 로직 소거:** UX 저하 및 사용자의 자율성을 침해할 소지가 있는 강제 화면 잠금, 5분 대기 타이머 등의 기능이 코드베이스 일체에서 삭제되었습니다.
*   **[추가] Conversational Nudge 구조 도입:** 강제 팝업 대신, 자연스러운 대화 맥락 속에서 프리미엄 링크를 넌지시 추천하는 '채팅 기반 큐레이션'으로 전환하여 클릭률과 유저 만족도를 동시에 잡습니다.
*   **[추가] 실전 팩트(Fact) 체커 도입:** AI의 고질적인 한계인 '정적 데이터 앵무새' 현상을 해결하기 위해, 질문 즉시 라이브 웹에 접근하여 최신 증시와 뉴스를 긁어오는 **Tool Calling** 모듈이 성공적으로 탑재되었습니다.
