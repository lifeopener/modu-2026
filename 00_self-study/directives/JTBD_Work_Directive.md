# 📜 JTBD 인터뷰 분석 및 보고서 작성 지침 (SOP)

이 문서는 모든 JTBD 인터뷰 산출물 작성 시 반드시 준수해야 할 표준 방법론과 보고서 양식을 정의합니다.

---

## 1. 분석 핵심 프레임워크: JTBD (Jobs-to-be-Done)
- **목표:** 고객이 특정 솔루션을 '고용한(Hire)' 근본적인 동기와 상황적 맥락(Context)을 파악하여 제품 전략으로 연결.
- **분류 체계:** 매수/매도 결정을 내리는 상황(Situation), 해결하려는 과업(Job Statement), 도달하려는 결과(Desired Outcome).

---

## 2. 인터뷰 결과 보고서 표준 필드

| 필드 | 작성 가이드 |
| :--- | :--- |
| **Persona / Segment** | **코어·확장·극단·비활성** 중 분류 + 짧은 설명(직무/맥락) |
| **Situation (When …)** | 전환을 유발한 "상황적 맥락" (한 문장) |
| **Job Statement** | 사용자가 "해결하려는 핵심 일(진보)" (한 문장) |
| **Desired Outcome** | **달성하고 싶은 결과 (측정 가능한 지표 중심)** |
| **4 Forces** | Push(불만), Pull(매력), Habit(관성), Anxiety(불안) 상세 분석 |
| **Current Solutions** | 현재 쓰는 대체수단 (툴/사람/엑셀/수기 등) |
| **Switch Triggers** | 채택/이탈을 유발한 결정적 계기와 장애요인 |
| **Evidence** | 인터뷰 인용구 (Quotes) 및 로그/지표 근거 |
| **Priority (AOS / DOS)** | 하위 산식을 이용한 객관적 지수 도출 |

---

## 3. 정량적 우선순위 산출 수식 (ODI)

### **AOS (Agility Opportunity Score)**
- **정의:** 중요도 대비 현재의 만족도 미달 수준 (높을수록 기회)
- **수식:** `AOS = 중요도(Importance) × (1 - 만족도(Satisfaction) / 5)`

### **DOS (Desired Outcome Score)**
- **정의:** 시장 규모를 고려한 비즈니스 파급 효과
- **수식:** `DOS = (중요도 - 만족도) × TAM(%)`

## 4. JTBD 요약 카드 및 Outcome 목록 표 양식
모든 보고서의 끝에는 위 지표가 포함된 **'JTBD 요약 카드'**와 **'Outcome 목록 표'**를 반드시 첨부하여 의사결정의 근거로 활용합니다.
