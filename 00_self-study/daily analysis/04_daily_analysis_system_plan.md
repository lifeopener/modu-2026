# 📊 매일 일관된 투자 분석을 받기 위한 전략 가이드

> **핵심 문제:** AI한테 "뭐 사야 해?"라고 물으면 **매번 다른 답**이 나온다.
> **근본 원인:** 그건 AI의 "의견(Opinion)"이지 "분석(Analysis)"이 아니기 때문.
> **해결책:** 데이터 기반 자동화 시스템을 만들어서, **숫자가 말하게** 하면 된다.

---

## 1. 왜 AI 의견은 매번 달라질까?

| 문제 | 설명 |
|---|---|
| **확률적 출력** | LLM은 매번 다른 토큰을 생성함. 같은 질문에도 다른 답이 나옴 |
| **컨텍스트 부재** | 이전 대화를 기억하지 못하므로 일관성 유지 불가 |
| **정량적 근거 부족** | "느낌"으로 추천하지, RSI·MACD·ADX 등 숫자를 보고 판단하지 않음 |
| **모델별 차이** | Gemini와 Opus는 다른 AI → 같은 데이터도 다르게 해석 |

---

## 2. 해결 방법: 3단계 자동화 시스템

### ✅ 이미 가지고 있는 것 (chart-analysis-agent/)

| 항목 | 파일 | 상태 |
|---|---|---|
| 7-Layer 분석 방법론 | `chart-analysis-prompt.md` (578줄) | ✅ 완성됨 |
| 기술적 분석 엔진 | `execution/chart_analyzer.py` (1090줄) | ✅ 완성됨 |
| 개별 지표 방법론 | `methodology/` (ADX, RSI, MACD, 볼린저 등 7개) | ✅ 완성됨 |
| 웹 서버 | `execution/app_server.py` | ✅ 기초 완성 |

### 🔧 추가로 만들어야 할 것

```
[STEP 1] 매일 아침 자동 실행되는 "데일리 브리핑 스크립트"
  └─ 입력: 관심 종목 리스트 (예: MSFT, NVDA, JPM, VOO, SPY)
  └─ 처리: yfinance 실시간 데이터 → 7-Layer 기술적 분석 자동 실행
  └─ 출력: 각 종목별 매수/매도/관망 스코어카드 (숫자 기반, 의견 아님)

[STEP 2] 매크로/뉴스 크롤러
  └─ 입력: 웹 검색 (Fear & Greed Index, PPI/CPI 일정, 어닝 캘린더)
  └─ 처리: 오늘 시장에 영향 줄 핵심 이벤트 자동 수집
  └─ 출력: 매크로 환경 요약 리포트

[STEP 3] 종합 의사결정 리포트 자동 생성
  └─ STEP 1(기술적 분석) + STEP 2(매크로) 결합
  └─ 일관된 포맷의 데일리 리포트 자동 생성
  └─ 매일 같은 기준, 같은 포맷 → AI 의견이 아닌 "데이터의 판단"
```

   → 기존 chart_analyzer.py를 바로 실행하여 오늘자 실시간 분석 결과 확인

   → 스크립트 + 알림 봇까지 풀 파이프라인 구축
